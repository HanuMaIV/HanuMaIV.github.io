What is Ruby on Rails?

Ruby on Rails is an open source web application framework based in the "Ruby" language. Rails was created by David Heinemeier Hansson. It was extracted from his company's work on Basecamp (a project management web application) and released to the public in 2004. Ruby is based on the MVC (model, view, controller) design pattern. Rails is flexibly enough to be used on many different server and database configurations.

MVC

Model–view–controller (MVC) is a software architectural pattern for implementing user interfaces. The software application is divided into 3 separate interacting components (model, view and controller). These components represent the way that data is received or presented to the web application user.

Model

The model component manages the data, logic and rules of the application.

View

Handles how the output is presented to the user.

Controller

Routes input and out from the user to the model or view components.

Some of the core principles in the Rails framework architecture:

  • Don't repeat yourself (DRY)
  • Convention over configuration
  • Everything is an object
  • Fat models, skinny controllers

Don't repeat yourself (DRY)

The DRY principle is stated as “Every piece of knowledge must have a single, unambiguous, authoritative representation within a system.” Rails provides the structure to effectively integrate this principle into your web application.

Convention over configuration

Convention over configuration is a principle in which a developer only needs to specify unconventional aspects of his web application. This reduces the amount of configuration needed from a base install web application while also allowing flexibility for added complexity.

Everything is an object

In Ruby, everything is an object. Every bit of information and code can be given their own properties and actions. Constructing software in this manner is called object-oriented programming. The Ruby Language was implemented with these principles taking precedence. Object-oriented programming implements properties by the name of "instance variables" and BEHAVIORS are known as "methods."

Fat models, skinny controllers

The Model and Controller aspects of rails should work well together. Static logic should go in the model. The “skinny” controller is the interface between the view and model.
