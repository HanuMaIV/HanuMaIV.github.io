What's the difference between an array and a hash?

In ruby Arrays are ordered, integer-indexed collections of any object.

When programming we first learn to store single pieces of information into variables in a way such as:

  • eye_color = brown
  • takes_showers = true
  • age = 33

    • This is all fine and dandy but what about when we want to hold more than one piece of information? Here comes the simple array.

    Simple Array

    Simple arrays can contain more than one piece of information. They do this by assigning a key and value to every element in the array.

    Let's create an array and take a closer look:

    new_array = ["brown", true, 33]

    Here we can see that arrays are created within [] brackets with the values separated with comas. What we can't really see here is the key. The handy chart below shows the key and values pairs for the new_array that we have created.

    new_array
    Key Value
    0 "brown"
    1 true
    2 33

    We can see that the values are paired with a key. The keys in this simple array are just integer values starting at 0. Meaning that the first value of the array will start at 0.

    We can access our values in the array by referencing the integer index in brackets following the array name as shown below:

    new_array[0]
          => "brown"
    new_array[1]
          => true
    new_array[2]
          => 33

    Great now we have a handy way of storing and accessing more than one element in a data structure.

    But indexes of [0], [1], [2], [3]..... don't really tell us much about the value of our data. What if we'd like a more descriptive way to index our values into keys?

    Hash (associative array) to the rescue

    In ruby we can associate our values to keys in a more descriptive way by using hashes.

    Let's create one now:

    new_hash = {"eye_color" => "brown",
            "takes_showers" => true,
            "age" => 33}

    Here we have successfully created a hash. You can see the hash keys and values are enclosed in parenthesis {}. The keys here are enclosed in quotes "" followed by a => then the corresponding value.

    Let's take a closer look at how the key and value pairs are indexed.

    new_hash
    Key Value
    "eye_color" brown
    "takes_showers" true
    "age" 33

    Wow that was simple. Now we can see that our key names contain more information about the value that they are associated with. So much better than 1,2,3,4 etc...

    We can access our hash values in a similar way to simple array but instead of using an index number we reference the key name enclosed in quotes.

    new_hash["eye_color"]
            => "brown"
    new_hash["takes_showers"]
            => true
    new_hash["age"]
            => 33

    Wow very simple and also might I say that based on the data provided that new_hash seems like quite the catch.

    Summary

    To recap:

    • Simple Arrays and Hashes both contain key and pair values.
    • Simple Array keys are indexed using integers starting at 0 and going up depending on how many items are added to the array.
    • Hash keys can be more descriptive about the value they are associated with.
    • Hash keys are indexed using a user inputed string or symbol.

    Just as a closing thought this article discusses arrays and hashes in the context of the ruby language. Further studies can reveal that both Simple Arrays and Hashes (associative arrays) can be considered a type of array. What??? what???? what??????
